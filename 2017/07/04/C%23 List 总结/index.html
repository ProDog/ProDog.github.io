C# List <T> 用法总结

C#
  • C#

C# 中 List 在 System.Collections.Generic 命名空间下。

为什么会有 List？它是做什么的？

首先，C# 中有数组 Array，数组中的数据连续存储，访问速度也最快，但是往数组中间插入数据就不方便了，另外，在声明数组时必须指定数组的长度和数据类型，使用时数组长度过长会造成资源浪费，数组长度过短则会溢出，而且当遇到一些不确定长度的数据处理问题时就会很麻烦。为了克服数组的这些不足，就有了 ArrayList，ArrayList 在 System.Collections 命名空间下，也继承了 IList 接口，ArrayList 可以根据存储的数据大小来动态扩展和收缩对象大小，声明时不需要指定长度，也不需指定数据类型，这就意味着 ArrayList 中可以存储不同的数据类型的数据，ArrayList 会把所有数据当做 object 类型来处理，这样使用简单的同时也导致了它不是类型安全的，类型的不统一也导致它在存储和检索数据时需要进行装箱和拆箱操作，而装箱拆箱操作又是十分损耗性能的。所以 List 就出现了， List 类是 ArrayList 类的泛型等效类。该类使用大小可按需动态增加的数组实现 IList 泛型接口。泛型使它为使用 C# 语言编写面向对象程序增加了极大的效力和灵活性，不会强行对值类型进行装箱和拆箱，或对引用类型进行向下强制类型转换，所以性能得到提高。List 的用法和 ArrayList 类似，只不过在使用 List 时需要指定数据类型。

数组容量固定，可以有多个维度；ArrayList 或 List 的可根据需要自动扩充、修改、删除或插入数据，只有一个维度。特定类型（Object 除外）的数组 的性能优于 ArrayList 的性能。 这是因为 ArrayList 的元素属于 Object 类型；
在决定使用 List 还是使用 ArrayList 类（两者具有类似的功能）时，List 类在大多数情况下执行得更好并且是类型安全的。如果对 List< T> 类的类型 T 使用引用类型，则两个类的行为是完全相同的。如果对类型 T 使用值类型，则需要考虑实现装箱问题。

添加到 ArrayList 中的任何引用或值类型都将隐式地向上强制转换为 Object。如果项是值类型，则必须在将其添加到列表中时进行装箱操作，在检索时进行取消装箱操作。强制转换以及装箱和取消装箱操作都会降低性能；在必须对大型集合进行循环访问的情况下，装箱和取消装箱的影响非常明显。
