比特币分叉到底怎么分？

比特币
  • Bitcoin

关心数字货币的人都知道，最近关于比特币最重大的事件就是 8 月的分叉了，这次分叉对于比特币的发展是至关重要的，但是只作为投资者的话怎么了解比特币分叉，分叉是什么，为什么要分叉，分叉后会变成两个比特币吗，这类问题对于很多人来说并不好理解，关于分叉的说明也全是专业名词，所以我花了点时间去了解比特币分叉相关知识，在这里做一下总结和介绍，尽量将分叉描述的易于理解。

为什么要分叉？

首先你要了解基本的比特币原理和区块链知识，比特币就是运行在全世界各个节点上的区块链大账本，每一笔比特币的转账交易都会被矿工记录并打包进一个区块中，所有区块按产生的时间顺序连在一起形成比特币的区块链。比特币的区块大小是有限制的，目前是 1M，按照平均 10 分钟产生一个区块计算，目前 1M 区块的大小可以支持的交易次数是每秒 3 笔，随着比特币的普及，这个数字显然满足不了比特币正常交易，所以我们需要一种办法来增加每秒处理的交易次数，这种方法可以是增大区块大小，或者减少需要记录进区块的交易数。

为什么一开始不把区块大小设置成我们想要的大小而是 1M 呢？在比特币刚出现的时候，比特币价格十分便宜，每个人都可以购买大量的比特币来玩，可以在各个地址间随意转让比特币，所以会导致很多没有意义的转账出现，或者有人恶意制造大量的垃圾交易来堵塞比特币网络，这时候比特币的发明者中本聪就将区块大小限制在 1M，防止垃圾交易出现，但是中本聪肯定也考虑到 1M 区块以后不够用，因为他相信比特币会普及，所以他设定的 1M 限制是临时的，同时也为以后增加容量做了准备，他在比特币源码中写下了相应的规则：当区块高度（就是区块链的长度）大于某一值时，区块容量扩大到某一个值。

但是你以为这样就解决问题了吗？不，因为代码是由人维护的，在中本聪消失后（关于他为什么消失以及去了何处都无人所知，他只是想改变世界确不想出名），他将维护工作交给了 Gavin，Gavin 又把代码维护权限分给其他开发者，后来，当区块高度快要达到中本聪设定的高度时，开发团队就是否要按照最初的设计进行扩容产生了分歧。Gavin 坚持按中本聪的设定来，其它开发者（Core 团队）认为有更好的方法可以替代中本聪当时的设定，这个方法就是隔离见证（SegWit），SegWit 类似于在比特币网络上增加一层网络，大部分交易走第二层快速网络，少数高价值结算交易走原始的第一层网络，类似于打牌记账，每次只记录转账数值，并不进行真正的转账，等最后结算时在进行正式的转账。SegWit 一般由类似交易所的机构来实现。Gavin 认为 SegWit 相当于改变了比特币原有的架构，对于系统的稳定性不利，而且政府如果打击比特币的话类似交易所的机构很容易被查封，不利于比特币的正常运行；
而 Core 团队认为按照中本聪的设计进行扩容的话，会导致区块无限增大，2M，4M，8M，最终会导致普通用户的电脑无法保存比特币全部区块，即运行全节点钱包，只有少数节点运行全节点钱包，这相当于损失了一部分去中心化的特点，不符合比特币的初衷，另外，关于中本聪设定的扩容方案，这是一种硬分叉方案，即扩容完成以后用户的钱包必须是扩容后的钱包，旧版钱包不兼容，这样相当于产生了一新一旧两种比特币，容易导致系统混乱。。关于全节点钱包的问题中本聪其实早就想到了，他在比特币的白皮书中用摩尔定律论证了计算机硬件发展速度快于比特币区块的增加速度，任何时候主流的电脑都可以满足全节点钱包的使用；而且中本聪还设计了垃圾回收机制，即可以删除很久远的区块，只保留最近的区块便可以确保安全。

最终观点还是不能统一，Core 开发者将 Gavin 等人踢出了原始开发团队，删除了他们的代码维护权限。

之后双方开发团队就如何进行比特币不断争论，希望能达成共识，这就有了比特币的纽约共识和香港共识等。最后一次共识便是香港共识，双方达成扩容到 2M 区块，并实现 SegWit，但是最终也以扩容方团队 BU 不认可而破裂。

那么现在进行的是哪种分叉？

SegWit 是最初由 Bitcoin Core 开发团队提出的向后兼容协议升级。它是 Bitcoin Core 支持的缩放路线图的核心，因为协议升级是在2015年12月首次提出的，它在当今网络上的许多有源比特币节点上实现。SegWit现在也是“ 纽约协议 ”的一部分：在许多比特币公司（包括许多矿工）之间建立的替代扩展路线图。而Bitcoin Improvement Proposal 148或BIP148，一个用户激活的软叉（UASF）计划在8月1日，也打算激活SegWit。

区别在于激活如何发生。

第一个和原来的选择是由比特币核心开发团队提出的。他们的SegWit代码由BIP141定义，激活了在大约两个星期的单个困难时期内，95％的哈希能力在11月15日之前表明准备就绪。假设表示准备的矿工实际上准备好支持升级，比特币的封锁和货币分割的风险很小。

然而，目前只有大约40％到45％的哈希功率是BIP141的信号准备。这就是为什么一些比特币用户计划用BIP148 UASF激活SegWit的原因。从8月1日开始，他们的节点将拒绝所有不符合BIP141要求的块。如果这个建议得到绝大多数矿工的支持（哈希力量），这些矿工应该总是声明最长的有效链，这个链应该在所有SegWit就绪的Bitcoin客户端上激活SegWit，并避免分裂。但是，如果这项建议只得到少数矿工的支持，那么“BIP148”链就可以从目前的议定书中解脱出来。

纽约协议也被称为“Silbert协议”或“SegWit2x”，计划通过BIP91激活SegWit。像BIP148一样，所有BIP91节点将拒绝任何不能使BIP141通知的块。但与BIP148不同的是，在大约两天的时间内，BIP91节点只能在80％的散列功率信号上支持BIP91。这也应尽量减少拆分的机会。

然而，“纽约协议”的第二步是比特币“基地块大小”的一个难题。这种变化与分离证人本身相反，并不是向后兼容的，因此也可能导致“分裂”。

还有一个这样的事情，矿工表示意图支持“纽约协议”，但这是有意义的，为了本文的目的将被忽略。

7月14日：BTC1部署
纽约协议产生的软件客户端是Bitcoin Core代码库的一个分支，称为“BTC1”.BTC1的beta软件于6月30日发布。但根据SegWit2x 路线图，7月14日是“纽约协议”签署人实际安装BTC1 beta软件并自行测试的日子。

然而，这不应该影响一般用户或矿工。

7月21日：BIP91信号开始

SegWit2x开发团队的目标是7月21日是BTC1节点实际运行的日子，重要的是矿工信号应该开始的那一天。

这也不应该真正影响普通用户。

但是，如果您是矿工，您可能希望通过信号通知BIP91来帮助激活SegWit。（这是通过挖掘“bit 4”块来完成的;例如，通过BTC1挖掘或使用包含BIP91补丁的Bitcoin软件）。

7月23日（最早）：BIP91锁定

BIP91激活的精确阈值要求在预定义的336个块中，269个块必须指示准备状态。这是大约80％的哈希能力，超过约2/3天。所以假设遵循SegWit2x路线图，BIP91可以在7月23日锁定。

再次，这不应该真正影响普通用户。

但如果您是矿工，并且BIP91确实激活，您现在还有一两天要遵守BIP91软叉。如果不这样做，就冒险无效块开采。

7月25日（最早）：BIP91激活

7月25日，BIP91“锁定”后又有336个块，BIP91可以在最早实际生效。任何不表示“隔离见证人”（按BIP141）的方块现在将被拒绝。

这还不能真正影响普通用户。

但是，如果你是矿工，你现在应该只有矿区，只有我的最高块，这就意味着SegWit（BIP141）。否则，您有可能通过哈希功能多数拒绝您的程序块。

7月29日：BIP91截止日期

如果比特币矿工想避免在比特币的块状和货币“分割”，7月29日是他们的第一个截止日期。

]为确保BTC1的BIP91及时启用以与BIP148兼容，应在最近一天（例如08:00 UTC前）锁定。（美国东海岸上午4点）。但是越早越好：BIP91锁定的最后期限越接近时间不会激活的风险就越大。

如果BIP91在此期限内未激活，Bitcoin可能会进行连锁分拆。

如果发生这种情况，如果您是普通用户，并且您还没有准备好潜在的BIP148 UASF连锁分拆，那么现在有两天的时间。

如果BIP91在7月29日之前没有启动，而您是矿工，那么现在有两天的时间可以决定您将在8月1日在BIP148连锁店或原始（“Legacy”）链路上将哪个连锁店。

7月31日：BIP141截止日期

从技术上讲，7月31日是矿工第二个截止日期避免分裂。

在这一天（UTC），最新的BIP91必须激活，或BIP141必须锁定。换句话说，如果BIP91没有及时锁定，现在必须有两个星期的困难时期，其中95哈希功率信号的百分比支持SegWit。

实际上，七月三十一号的机会很小，实际上与困难时期的结束日相吻合。即使如此，直到今天才明白是否达到95％的门槛是不可能的。

因此，如7月29日所述，用户和矿工都应该已经准备好潜在的分拆。但7月31日真的是你最后准备好的机会。

8月1日：BIP148激活

这是BIP148激活的一天，这使得矿工避免分裂的第三个也是最后截止日期。

8月1日，在00:00 UTC（请注意，这仍然是美国7月31日），所有BIP148节点将开始拒绝任何不表示“隔离见证人”（按BIP141）准备的块。如果Bitcoin Core的BIP141或BTC1的BIP91已经及时地锁定和/或激活，那么应该没有链式拆分。

如果两者都没有发生，大多数矿工（通过哈希力）现在有最后的机会避免连锁分裂：通过支持BIP148本身。这将确保他们始终（重新）根据所有当前比特币节点声明最长的有效链，并将通过BIP141激活SegWit。

如果BIP141，BIP91和BIP148在8月1日之前已经吸引了足够的散列功率，但是BIP148确实获得了一些牵引力，这一天链可能会分裂。如果发生这种情况，将会有两种类型的“比特币”，我们称之为“148比特币”或“148BTC”为BIP148侧的分割，而“旧的比特币”或“LegacyBTC”为另一方。（如果148 Bitcoin收回了最长的链，这种分裂可能会随着时间的推移而解决）

如果您在分裂时拥有比特币（即私钥），则应自动拥有148BTC和LegacyBTC。虽然，要访问这两种类型，您可能需要下载新的软件。也就是说，如果您是常规用户，最好暂停任何和所有交易，至少直到情况更清楚。没有这样做可能会花费你的钱！

如果分裂发生，你是一个矿工，你现在应该决定在哪个链上挖矿。（当然，您仍然可以随时切换，例如，您可能只想挖掘最有利可图的链条。）

8月4日：潜在的比特币ABC发布

虽然确切的日期并没有确定，但8月4日可能会看到引入一个新的“比特币”：比特币ABC。

主要比特币矿业硬件生产商Bitmain 最近 宣布，如果BIP91在8月1日之前没有激活，BIP148在8月1日之后出现生命迹象，将会启动“应急计划”。最近，实施这一应急计划 - 名为“比特币ABC“ - 在比特币会议的未来宣布。

Bitcoin ABC将与Bitcoin共享历史，直到8月1日，但在之后的头几天，可能是由Bitmain私人开采的。它还将包括几个新的和不兼容的协议规则，也许最重要的是增加块大小限制。

如果您在分割时拥有比特币 - 8月1日，00:00 UTC - 您应该自动拥有“比特币ABC”。但是，在这种情况下，您可能需要下载软件才能发送（并接收）硬币。如果发生此事件，更多信息可能会变得可用。

如果你是一个矿工，你应该能够在8月4日开始使用比特币ABC。（也许早一点）

8月15日：潜在的BIP148工作证明变更
虽然这个日期还没有确定，但8月15日可能会看到另一种类型的“比特币”的引入，我们将其称为“新派比特币”或“新宝贝”。

作为UASF，8月1日的BIP148叉头最初是由用户实现的 - 而不是矿工。用户将开始拒绝任何不表示“隔离见证人”（根据BIP141）准备就绪的块。这应该鼓励矿工遵循BIP148连锁（理想情况下是唯一的链条）。但这是否真的会发生呢还有待观察。到目前为止，很少有矿工公开表示会支持UASF。

如果对148位比特币的矿工支持仍然很低，（其中一些）用户可能会选择实施硬叉来更改工作证明算法。这将通过使其专门的ASIC采矿硬件在分拆的BIP148侧淘汰，从而“开火”现有的矿工，并可能为新的矿工敞开大门 - 可能会将采矿景观重新分散回家用户与CPU或GPU 。（至少暂时）

在“ 比特币核心社区 ”Slack 的子频道上，一个开放的BIP148“工作组” 目前正在拟定拟议的细节。但是似乎如果8月15日在148比特币的哈萨克斯坦矿山总数不到15％，这个硬盘将被部署。这将使与“传统比特币”的分裂确定。如果某些用户拒绝加入改变工作验证算法的硬分支，那么技术上也可能导致148比特币方面的另一个分裂。

如果您是8月1日分拆时的常规用户和拥有的比特币，并且直到8月15日才支付任何费用，那么您现在还应该拥有NewPoWBTC。如果您在8月1日至8月15日期间购买或获得了148BTC，您现在应该也拥有NewPoWBTC。

但是，如果你在分裂的任何一边都用比特币，那就不一定了。再次，如果你想要安全的一面，一切都不要交易，直到情况变得更加清楚。（有关详细信息，请参阅这篇文章。）

如果您是使用ASIC硬件在148位比特币上挖掘的少数矿工之一，您可能现在想将硬件指向其他地方，或完全关闭硬件。（这当然取决于我的148BTC或其他任何东西是否还有利可图。）

当然，任何人现在都可以在没有专门的ASIC硬件的情况下在NewPoWBTC上开采。

8月中旬至8月：SegWit锁定

如果矿工避免通过BIP141，BIP91或BIP148进行连锁分拆，则隔离证人应该在8月中旬或8月下旬之间锁定。这应该是一个非事件，如果它发生通过BIP91或BIP148，因为它将在这一点上预期，并不会改变任何人。

但是锁定手段是所有SegWit准备好的客户将在下一个困难时期开始执行新的规则：再过两个星期。因此，如果您是常规用户，并希望使用新功能或想要保持一流的安全性，则现在有两周时间升级到强制执行SegWit的软件。

如果没有避免连锁分裂，情况就会变得复杂得多。到8月中旬，可能会有一个，两个，三个甚至四个“比特币”：“148比特币”，“新派比特币”，“比特币ABC”以及“旧版比特币”。

在撰写本文时，很难说如何处理这种情况。对于普通用户来说，最好是按住你的私钥，最好不要一直处理，直到情况以某种方式解决。

对于矿工来说，最好是挖掘最有利可图的链条。

8月下旬至9月中旬：SegWit激活

如果避免链式拆分，SegWit应该在这段时间内激活。如果您现在正在运行兼容的客户端，您可以开始享受新功能，同时保持一流的安全性。

如果没有避免连锁分裂，情况可能仍然太复杂，无法预测。

十月下旬至十一月中旬：SegWit2x硬叉

到今年年底（尽管可能甚至到2018年初），SegWit2x的硬叉部分应该发生。确切地说，在SegWit锁定三个月后，这个硬盘在BTC1客户端上激活。这个激活意味着应该开采大于1兆字节的“基本块”，这使得BTC1客户端与所有没有硬性的所有Bitcoin客户端不兼容叉代码实现。

在这个时候，似乎几乎肯定不是每个人都会改变他们的软件来支持这个硬盘。比特币的发展界尤其几乎无动于衷地 拒绝了这一建议，而几家公司表示不愿意支持这一改变，而不是所有的用户似乎都在板上。因此，这可能导致另一个连锁分裂。（理论上来说，这可能会导致第五个“比特币”，但是看起来似乎不太可能会有四个“比特币”。）

如何处理这种情况，普通用户将在比特币杂志上公布更接近实际的日期。

如果您是矿工，而您只运行BTC1客户端，以帮助激活SegWit而无意支持硬叉，那么您应该确保在此之前切换回非BTC1客户端。
