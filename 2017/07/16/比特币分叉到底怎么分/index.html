比特币分叉到底怎么分？

比特币
  • Bitcoin

关心数字货币的人都知道，最近关于比特币最重大的事件就是 8 月的分叉了，这次分叉对于比特币的发展是至关重要的，但是只作为投资者的话怎么了解比特币分叉，分叉是什么，为什么要分叉，分叉后会变成两个比特币吗，这类问题对于很多人来说并不好理解，关于分叉的说明也全是专业名词，所以我花了点时间去了解比特币分叉相关知识，在这里做一下总结和介绍，尽量将分叉描述的易于理解。

为什么要分叉？

首先你要了解基本的比特币原理和区块链知识，比特币就是运行在全世界各个节点上的区块链大账本，每一笔比特币的转账交易都会被矿工记录并打包进一个区块中，所有区块按产生的时间顺序连在一起形成比特币的区块链。比特币的区块大小是有限制的，目前是 1M，按照平均 10 分钟产生一个区块计算，目前 1M 区块的大小可以支持的交易次数是每秒 3 笔，随着比特币的普及，这个数字显然满足不了比特币正常交易，所以我们需要一种办法来增加每秒处理的交易次数，这种方法可以是增大区块大小，或者减少需要记录进区块的交易数。

为什么一开始不把区块大小设置成我们想要的大小而是 1M 呢？在比特币刚出现的时候，比特币价格十分便宜，每个人都可以购买大量的比特币来玩，可以在各个地址间随意转让比特币，所以会导致很多没有意义的转账出现，或者有人恶意制造大量的垃圾交易来堵塞比特币网络，这时候比特币的发明者中本聪就将区块大小限制在 1M，防止垃圾交易出现，但是中本聪肯定也考虑到 1M 区块以后不够用，因为他相信比特币会普及，所以他设定的 1M 限制是临时的，同时也为以后增加容量做了准备，他在比特币源码中写下了相应的规则：当区块高度（就是区块链的长度）大于某一值时，区块容量扩大到某一个值。

但是你以为这样就解决问题了吗？不，因为代码是由人维护的，在中本聪消失后（关于他为什么消失以及去了何处都无人所知，他只是想改变世界确不想出名），他将维护工作交给了 Gavin，Gavin 又把代码维护权限分给其他开发者，后来，当区块高度快要达到中本聪设定的高度时，开发团队就是否要按照最初的设计进行扩容产生了分歧。Gavin 坚持按中本聪的设定来，其它开发者（Core 团队）认为有更好的方法可以替代中本聪当时的设定，这个方法就是隔离见证（SegWit），SegWit 类似于在比特币网络上增加一层网络，大部分交易走第二层快速网络，少数高价值结算交易走原始的第一层网络，类似于打牌记账，每次只记录转账数值，并不进行真正的转账，等最后结算时在进行正式的转账。SegWit 一般由类似交易所的机构来实现。Gavin 认为 SegWit 相当于改变了比特币原有的架构，对于系统的稳定性不利，而且政府如果打击比特币的话类似交易所的机构很容易被查封，不利于比特币的正常运行；
而 Core 团队认为按照中本聪的设计进行扩容的话，会导致区块无限增大，2M，4M，8M，最终会导致普通用户的电脑无法保存比特币全部区块，即运行全节点钱包，只有少数节点运行全节点钱包，这相当于损失了一部分去中心化的特点，不符合比特币的初衷，另外，关于中本聪设定的扩容方案，这是一种硬分叉方案，即扩容完成以后用户的钱包必须是扩容后的钱包，旧版钱包不兼容，这样相当于产生了一新一旧两种比特币，容易导致系统混乱。。关于全节点钱包的问题中本聪其实早就想到了，他在比特币的白皮书中用摩尔定律论证了计算机硬件发展速度快于比特币区块的增加速度，任何时候主流的电脑都可以满足全节点钱包的使用；而且中本聪还设计了垃圾回收机制，即可以删除很久远的区块，只保留最近的区块便可以确保安全。

最终观点还是不能统一，Core 开发者将 Gavin 等人踢出了原始开发团队，删除了他们的代码维护权限。

之后双方开发团队就如何进行比特币不断争论，希望能达成共识，这就有了比特币的纽约共识和香港共识等。最后一次共识便是香港共识，双方达成扩容到 2M 区块，并实现 SegWit，但是最终也以扩容方团队 BU 不认可而破裂。

那么现在进行的是哪种分叉？

SegWit 是最初由 Bitcoin Core 开发团队提出的向后兼容协议升级。它是 Bitcoin Core 支持的缩放路线图的核心，因为协议升级是在2015年12月首次提出的，它在当今网络上的许多有源比特币节点上实现。SegWit现在也是“ 纽约协议 ”的一部分：在许多比特币公司（包括许多矿工）之间建立的替代扩展路线图。而Bitcoin Improvement Proposal 148或BIP148，一个用户激活的软叉（UASF）计划在8月1日，也打算激活SegWit。

区别在于激活如何发生。

第一个和原来的选择是由比特币核心开发团队提出的。他们的SegWit代码由BIP141定义，激活了在大约两个星期的单个困难时期内，95％的哈希能力在11月15日之前表明准备就绪。假设表示准备的矿工实际上准备好支持升级，比特币的封锁和货币分割的风险很小。
